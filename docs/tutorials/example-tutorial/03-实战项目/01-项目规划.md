# 3.1 项目规划

## 项目概述

我们将创建一个简单的待办事项（Todo）应用，用于演示完整的开发流程。

## 功能需求

### 核心功能

1. **添加任务**
   - 用户可以添加新的待办事项
   - 每个任务包含标题和描述

2. **查看任务**
   - 显示所有待办事项列表
   - 支持按状态筛选

3. **更新任务**
   - 标记任务为已完成
   - 编辑任务内容

4. **删除任务**
   - 删除不需要的任务

### 扩展功能

- 任务优先级
- 任务分类
- 搜索功能

## 技术选型

### 前端

- **框架**：Vue 3
- **UI 库**：Element Plus
- **状态管理**：Pinia

### 后端

- **框架**：Node.js + Express
- **数据库**：SQLite
- **ORM**：Sequelize

## 数据库设计

### tasks 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| title | VARCHAR | 任务标题 |
| description | TEXT | 任务描述 |
| status | VARCHAR | 状态（pending/completed） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## API 设计

### 获取任务列表

```
GET /api/tasks
Response: { tasks: [...] }
```

### 创建任务

```
POST /api/tasks
Body: { title, description }
Response: { task: {...} }
```

### 更新任务

```
PUT /api/tasks/:id
Body: { title?, description?, status? }
Response: { task: {...} }
```

### 删除任务

```
DELETE /api/tasks/:id
Response: { success: true }
```
