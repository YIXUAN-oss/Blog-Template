# 📖 教程中心使用指南

> 本指南将帮助你快速添加和管理教程内容

## 🚀 快速开始

### 1. 添加新教程分类

在 `docs/tutorials/` 目录下创建新的教程目录：

```bash
docs/tutorials/
└── your-tutorial/          # 你的教程目录名
    ├── README.md           # 教程首页（必需）
    └── 01-基础/            # 子分类目录
        ├── README.md
        └── 01-第一课.md
```

### 2. 创建教程首页

每个教程分类都需要一个 `README.md` 作为首页，示例：

```markdown
---
title: 你的教程名称
autoGroup: false
autoSort: false
sidebarDepth: 0
---

# 你的教程名称

> 教程简介和描述

## 📚 教程简介

这里介绍你的教程内容...

## 🎯 学习目标

- ✅ 目标1
- ✅ 目标2
- ✅ 目标3

## 📖 教程目录

### [第一章：基础入门](01-基础/01-第一课.md) ⭐⭐⭐⭐⭐
**学习内容：**
- 知识点1
- 知识点2
- 知识点3

### [第二章：进阶内容](02-进阶/01-进阶内容.md) ⭐⭐⭐⭐
**学习内容：**
- 知识点1
- 知识点2
```

### 3. 创建教程文章

在对应的子目录下创建 Markdown 文件：

```markdown
---
title: 教程文章标题
date: 2025-01-01
categories:
  - 教程分类
tags:
  - 标签1
  - 标签2
---

# 教程文章标题

## 章节内容

教程内容...
```

## 📝 在教程中心添加卡片

编辑 `docs/tutorials/README.md`，在教程卡片区域添加你的教程：

```html
<!-- 你的教程卡片 -->
<div class="magic-card" data-href="your-tutorial/">
  <span class="magic-card__bg"></span>
  <div class="card-header">
    <div class="card-icon">🎯</div>
    <span class="card-badge badge-hot">热门</span>
  </div>
  <div class="card-content">
    <h3>你的教程名称</h3>
    <p class="card-desc">教程简短描述</p>
    <div class="card-modules">
      <div class="module-title">📑 包含模块：</div>
      <div class="module-list">
        <div class="module-item">📚 模块1</div>
        <div class="module-item">📚 模块2</div>
        <div class="module-item">📚 模块3</div>
      </div>
    </div>
    <div class="card-footer">
      <span class="footer-text">开始学习 →</span>
    </div>
  </div>
</div>
```

### 卡片配置说明

- **`data-href`**: 教程目录路径（相对于 `docs/tutorials/`）
- **`card-icon`**: 教程图标（emoji 或文字）
- **`card-badge`**: 徽章样式
  - `badge-hot`: 热门（红色渐变）
  - `badge-recommend`: 推荐（蓝色渐变）
  - `badge-essential`: 基础/核心（黄色渐变）
  - `badge-frontend`: 前端（绿色渐变）
- **`card-desc`**: 教程简短描述
- **`module-item`**: 教程包含的模块列表

## 🎨 教程目录结构建议

### 推荐结构

```
your-tutorial/
├── README.md                    # 教程首页
├── 01-基础入门/
│   ├── README.md               # 章节首页（可选）
│   ├── 01-第一课.md
│   ├── 02-第二课.md
│   └── ...
├── 02-进阶内容/
│   ├── README.md
│   ├── 01-进阶内容1.md
│   └── ...
└── 03-实战项目/
    ├── README.md
    └── 01-项目实战.md
```

### 命名规范

- **目录名**: 使用数字前缀 + 描述，如 `01-基础入门`
- **文件名**: 使用数字前缀 + 标题，如 `01-第一课.md`
- **README.md**: 每个分类目录建议包含一个 README.md 作为索引

## 📸 图片管理

### 教程图片存储

在教程目录下创建 `attachments` 目录存放图片：

```
your-tutorial/
├── attachments/
│   ├── image1.png
│   └── image2.jpg
└── 01-基础/
    └── attachments/
        └── lesson1-image.png
```

### 图片引用

```markdown
<!-- 同级目录下的图片 -->
<img src="/tutorials/your-tutorial/attachments/image.png" alt="描述" style="border-radius: 8px;" />

<!-- 子目录下的图片 -->
<img src="/tutorials/your-tutorial/01-基础/attachments/image.png" alt="描述" style="border-radius: 8px;" />
```

> 💡 **提示**: 可以使用 `process_tutorial_images.py` 脚本自动处理图片样式

## ⚙️ 配置侧边栏（可选）

如果需要自定义侧边栏，编辑 `docs/.vuepress/config.ts`：

```typescript
series: {
  '/tutorials/your-tutorial/': [
    {
      text: '基础入门',
      children: [
        '/tutorials/your-tutorial/01-基础/01-第一课.md',
        '/tutorials/your-tutorial/01-基础/02-第二课.md',
      ]
    },
    {
      text: '进阶内容',
      children: [
        '/tutorials/your-tutorial/02-进阶/01-进阶内容1.md',
      ]
    }
  ]
}
```

## 💡 最佳实践

### 内容组织

1. **清晰的目录结构**: 使用数字前缀保持顺序
2. **完整的 README**: 每个教程分类都应该有详细的 README
3. **章节索引**: 在 README 中列出所有章节，方便导航

### 写作建议

1. **循序渐进**: 从基础到进阶，逐步深入
2. **代码示例**: 提供完整可运行的代码示例
3. **实践项目**: 包含实战项目，帮助巩固知识
4. **图片说明**: 为图片添加有意义的 alt 文本

### 维护建议

1. **定期更新**: 保持教程内容的最新性
2. **检查链接**: 定期检查内部链接是否有效
3. **用户反馈**: 根据用户反馈改进教程内容

## 🔧 常见问题

### Q: 如何让教程在首页显示？

编辑 `docs/README.md`，在教程中心卡片区域添加你的教程卡片。

### Q: 教程文章不显示在侧边栏？

检查 `docs/.vuepress/config.ts` 中的 `series` 配置，确保路径正确。

### Q: 图片不显示？

- 检查图片路径是否正确（使用绝对路径）
- 确保图片文件存在
- 检查文件名大小写

### Q: 如何添加代码高亮？

使用 Markdown 代码块，指定语言：

````markdown
```python
def hello():
    print("Hello, World!")
```
````
